## v0.0.29
Math mode. Emphasis. Build.

- [Bug] Fixed bold and italic markers with interior whitespace not rendering as emphasis. Many authors leave spaces inside emphasis markers (e.g. `*text *`, `** text **`, `_text _`) which CommonMark rejects because the delimiters are not "flanking". Added a `normalizeEmphasisWhitespace()` pre-processing step to the Markdown converter that relocates leading/trailing spaces from inside the delimiters to outside them (e.g. `** bold **` → ` **bold** `), preserving word separation while making delimiters flanking-compliant. Code blocks (fenced and inline) are protected from modification, and bullet list markers (`* item`) are detected so only the content after the marker is processed. Also relaxed the AsciiDoc converter's constrained bold/italic regex to allow interior whitespace, relocating it outside the HTML tags.
- Magazine featured lists now render articles full-width using the standard Card component, showing author and cover image. Category headings are larger and stick to the top of the viewport while scrolling through a category's articles. Removed the old 66/34 split layout.
- [Bug] Fixed highlights not appearing on article pages when the stored highlight coordinate used a different kind prefix than the article page expected. Highlights ingested from relays may have `a` tags referencing `30024:pubkey:slug` (draft) or other addressable kinds, while the article page always queries with `30023:pubkey:slug`. All highlight repository queries now match on the `pubkey:slug` suffix instead of requiring an exact coordinate string match, and the Redis cache key is normalized to `pubkey:slug` so that all kind variants share one cache entry.
- [Bug] Fixed math and code block recognition in the Markdown converter. Three issues were resolved: (1) Unclosed ` ```latex ` / ` ```math ` / ` ```tex ` fenced code blocks consumed all content to end-of-file, turning everything into one giant display math block — unclosed blocks now have their fence line removed so that any `$$…$$` sections inside are handled individually while prose between them is preserved. (2) Multi-backtick inline code spans (``` `` ``` , etc.) and unclosed fenced code blocks were not protected from math extraction, allowing `$$` inside code to be incorrectly treated as math delimiters — both are now properly shielded. (3) A lone unpaired `$$` (e.g. "Empty math: $$") could match with a distant `$$` across paragraphs of unrelated prose — display math matching now uses a two-pass approach: first matching `$$…$$` within single paragraphs (no blank-line crossings), then a second pass only for multi-paragraph blocks that contain actual LaTeX markup. Closed ` ```latex ` blocks whose content already contains `$$…$$` delimiters now strip the fence and output content as-is instead of double-wrapping.
- [Bug] Fixed inline math (`$...$`) not rendering in articles. KaTeX only recognizes `\(...\)` for inline math, not `$...$`. The server-side pipeline converts `$...$` to `\(...\)` via placeholders, but cached `processed_html` from before the placeholder system still contained raw `$...$` that KaTeX couldn't render — display math (`$$...$$`) worked because it was always in KaTeX's delimiter list. Added a client-side `normalizeDollarMathInTextNodes()` pre-processing step to both the `utility--katex` Stimulus controller and the UnfoldBundle `katex-init.js` that walks DOM text nodes and converts `$...$` to `\(...\)` before KaTeX runs. Currency values (`$10.50`, `$19.99`) and text inside `<pre>`, `<code>`, and other protected elements are skipped using the same TeX whitespace rule and heuristics as the server-side pipeline.
- [Bug] Fixed backtick-wrapped display math unwrapping regex crossing newlines and mangling article content. The regex that converts `` `$$…$$` `` to `$$…$$` used the `/s` flag, allowing `.+?` to match across newlines — when one `` `$$…$$` `` appeared on one line and another far away, the regex ate all content between them, causing MATHPH placeholders and CODEBLOCK tokens to leak into rendered HTML and triggering cascading KaTeX parse errors. Removed the `/s` flag so the regex only matches single-line backtick-wrapped display math.
- [Bug] Fixed display math `$$` matching across lines within the same paragraph. The single-backtick code protection regex treated backslash-escaped backticks (`\``) as real backtick delimiters, causing `$${amount}` in JavaScript template literals to be left unprotected. The exposed `$$` then paired with a distant `$$` (e.g. "Empty math: $$") across multiple lines, swallowing prose between them. Restricted the primary display math regex to single-line matching only (`/\$\$([^\n]+?)\$\$/`); multi-line display math blocks (like `\begin{cases}…\end{cases}`) are still handled by the secondary pass which requires LaTeX markup content.
- [Bug] Fixed inline code spans with backslash-escaped backticks rendering incorrectly. Content like `` `const price = \`$${amount}\`` `` was split by CommonMark at the first `\``, producing `<code>const price = \</code>` + bare `$${amount}` as text. CommonMark doesn't support `\`` escaping in code spans. Added a normalization step that detects single-backtick code spans containing `\`` and converts them to double-backtick syntax (`` `` const price = `$${amount}` `` ``), which CommonMark handles correctly. Fenced code blocks are protected from the conversion.
- [Bug] Fixed "Symfony Runtime is missing" fatal error on gateway (and other worker) services in dev. The docker entrypoint ran `composer install --no-scripts`, which suppresses the `POST_AUTOLOAD_DUMP` event that the `symfony/runtime` Composer plugin needs to generate `vendor/autoload_runtime.php`. Added an explicit `composer dump-autoload` step after sync so the runtime autoloader is always generated before any `bin/console` call.
- Faster dev container startup. Replaced the database readiness check (`php bin/console dbal:run-sql` — full Symfony kernel boot per attempt) with a lightweight raw PDO connection test (`docker/wait-for-db.php`, <1 s per attempt). Added a raw-SQL migration freshness check (`docker/check-migrations.php`) that compares file count vs applied count and skips the `doctrine:migrations:migrate` kernel boot entirely when migrations are already up-to-date. Replaced the recursive `setfacl -R` on the entire `var/` tree (thousands of cached files on bind mounts) with targeted ACLs on just the `var/`, `var/cache/`, and `var/log/` directories plus default ACL inheritance for new files.
- [Bug] Changed highlights grid to single-column layout at all screen sizes. Previously used a multi-column masonry layout (3 columns on large screens, 2 on medium) that made highlight cards too narrow to read comfortably.
- [Bug] Fixed LaTeX/KaTeX math rendering for articles using Nostr math markup conventions. Three issues were resolved: (1) Backtick-wrapped math (`` `$E=mc^2$` ``, `` `$$...$$` ``) was rendered as `<code>` elements that KaTeX skipped — a new `normalizeNostrMathMarkup()` step unwraps these before Markdown parsing. (2) Unclosed ` ```latex ` / ` ```math ` / ` ```tex ` fenced code blocks caused everything to EOF to become a code block — they are now converted to `$$...$$` display math. (3) CommonMark's backslash escape processing was destroying `\(`, `\)`, and `\\` in math expressions — the old `normalizeInlineMathDelimiters()` converted `$...$` to `\(...\)` in raw Markdown, but CommonMark treated `\(` as an escaped `(` and stripped the backslash, breaking inline math. Replaced with a two-phase placeholder system: `extractMathPlaceholders()` replaces all math (inline `$...$`, display `$$...$$`, `\(...\)`, `\[...\]`) with inert alphanumeric tokens before CommonMark, and `restoreMathPlaceholders()` re-injects the real math with delimiters intact into the HTML output after conversion. KaTeX's `ignoredTags` was also updated to allow rendering inside inline `<code>` elements (keeping `<pre>` ignored for fenced code blocks).
- Replaced synchronous relay fetching for embedded nostr references with deferred component-based resolution. The Converter no longer makes any relay calls — nostr references not found in the local DB are emitted as lightweight `<div class="nostr-deferred-embed">` placeholders in the cached `processed_html`. A new `resolve_nostr_embeds` Twig filter replaces each placeholder with a rendered `Molecules:NostrEmbed` Twig component (via `ComponentRendererInterface::createAndRender()`) that resolves the reference from DB/Redis in its `mount()` method. A `nostr--deferred-embed` Stimulus controller handles client-side fetch for anything still missing. This dramatically speeds up both batch `articles:process-html` and on-the-fly article rendering, and ensures `processed_html` is always saveable. The `NostrSchemeParser` CommonMark inline extension was simplified to never make relay calls.
- [Bug] Excluded preview/partial routes (`/editor/markdown/preview`, `/article-editor/preview/`) from visitor analytics. These are HTML fragment responses used by the editor, not real page views, and were inflating visit counts.
